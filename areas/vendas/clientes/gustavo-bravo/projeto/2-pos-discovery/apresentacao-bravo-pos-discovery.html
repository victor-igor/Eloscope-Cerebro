09 · O padrão por trás dos 3 gargalos

A mesma causa-raiz aparece 3 vezes.

Ausência de SLA contratual + ritual de medição diária.
Os 3 gargalos críticos têm causa-raiz comum. Resolver a raiz desbloqueia tudo.
5 Whys completo (aprovação)
  1. Por que aprovação demora 2-3 dias?
    → Cliente recebe no WhatsApp, mistura, esquece.
  2. Por que mistura?
    → Não tem canal dedicado nem prazo.
  3. Por que sem prazo?
    → Contrato não obriga.
  4. Por que contrato não obriga?
    → Bravo nunca formalizou.
  5. Por que nunca formalizou?
    → Cresceu sem processo (de 7 → 4 pessoas), virou pessoal.
A solução é em 2 camadas
1. Camada contratual (humana)
Cláusula de SLA — "se não responder em X dias, considera-se aprovado por decurso". Lucas redige modelo, Gustavo valida com advogado.

2. Camada técnica (automação)
OpenClaw + skills cobrando, agendando e analisando — time da Bravo opera as skills no dia a dia.
"Sem causa-raiz, automação só anestesia o sintoma e o problema volta em outro lugar."

11 · Decisão proposta

As 3 skills que vão entrar no escopo.

Cada uma ataca um gargalo crítico, dentro do OpenClaw da Bravo.
SKILL 1
Aprovação
com SLA

O que faz: envia pacote pro cliente via WhatsApp (n8n), monitora resposta, dispara lembretes 24h e 48h, escala pro Gustavo em 72h.

Stack: n8n + WhatsApp Business API + handoff JSON do Content Machine

Ganho: 2-3 dias → <24h em 70% dos casos

Bônus contratual: aditivo de SLA (Lucas redige).

SKILL 2
Agendador
multi-cliente

O que faz: recebe pacote aprovado + cronograma, agenda automaticamente em Meta/Instagram (futuro: TikTok/LinkedIn).

Stack: Meta Graph API + n8n + fila de jobs

Ganho: ~10h/mês manuais → <1h (só exceções)

Substitui: M-Labs / Reportei / Canva agendador.

SKILL 3
Análise
mensal

O que faz: dia 1-3 do mês roda análise: métricas Instagram + Analytics + benchmarks. Gera dossiê + recomendações.

Stack: Claude Code skill + handoff JSON pro Content Machine

Ganho: dado em vez de achismo + ~1h/cliente/mês economizada

Pendência: cronometrar análise real pra calibrar.

Onde os agentes ficam: dentro do OpenClaw da Bravo. Time abre, chama a skill, valida, segue. Não conversam com cliente final — única exceção: lembrete de cobrança automático na Skill 1.
