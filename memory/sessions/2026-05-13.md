## Decisões

- **elobrain é framework instalável, não embedded** — fork público mantém branding gbrain genérico, cerebros (Eloscope e clientes) consomem via MCP
- **Schema `brain` dedicado no Supabase** — isolamento total do `public` (evita conflito com `public.tags` da plataforma Elosystem que é usada na view `api_integration_contacts_v2`)
- **Skills com fallback defensivo** — funcionam com ou sem o framework instalado; permite rolar pra produção sem dependência hard
- **Connection pooler com `options=-c search_path=brain,public`** — funciona via pgbouncer, não precisa de SET por sessão
- **Vai virar produto Eloscope** — "Cerebro Claude Code by Eloscope" será pacote: template repo + INSTALL_ELOBRAIN.md + migrations + .env.example + .claude/settings.json pré-config
